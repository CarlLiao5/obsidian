WFS reports: Inbound receipts

Last updated on Feb 9, 2026

Reading time: 2 min

* * *

In this guide:

- [Overview](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-inbound-receipts#overview)
- [Access this report](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-inbound-receipts#accessthisreport)
- [Dive into details](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-inbound-receipts#dive_into_details)

* * *

Overview

You can use this report to better understand both the inbound receiving and the quantities for the orders you have created.

## Access this report

1. Log in to Seller Center.
2. Navigate to the [**WFS reports**](https://seller.walmart.com/wfs/reports) page.
3. Under **Inventory**, select _Download_ for the **Inbound receipts** report.
4. Set your desired time period.
5. Select _Download_ and open the report.

## Dive into details

### How to use this report effectively

- Select a time period or individual GTIN to include in the Inbound receipts report.

### Important insights

- Expected units can be compared to what was checked in at the Inbound Consolidation Center (if applicable), received or damaged units.
- This report is a great starting point to investigate inbound issues prior to submitting a dispute if needed.

## Understand report metrics

|     |     |
| --- | --- |
| **Column name** | **Description** |
| Checked in units at ICC | When Inventory is sent to a consolidation center, it is first checked in at the ICC before it flows downstream to FCs. This will indicate the units that have reached the ICC. |
| Damaged Units | Units that were received at the FC but were determined to be damaged by the FC associate |
| Description | Item description, defined during product setup |
| Expected Units | These are the units that were specified during PO creation |
| GTIN | 14-digit Global Trade Item Number, including check digit, that's unique to the item. If it's less than 14 digits, add zeros at the beginning |
| Inbound Order ID | Unique number assigned to the inbound order. An inbound order can contain one or more PO Number |
| Original PO Number | In the chance that a PO could not be received at the FC, an additional PO may be created.  If this is the case, the original PO will be listed. |
| PO Create Date | Date that the PO was created in seller center |
| PO Delivered Date | Date of when the first shipment for this PO was delivered |
| PO Number | Unique number assigned to a specific shipment |
| PO Status | Status in the PO will include:<br>Awaiting Delivery<br>Cancelled GDM<br>Processed GDM<br>Received |
| Received Units | Units when decanted and received at the FC |
| SKU | A unique item identifier provided during item setup, that represents the item purchased by the customer |
| UPC | A 12-digit number representing a standardized barcode used worldwide to identify products in retail settings. |
