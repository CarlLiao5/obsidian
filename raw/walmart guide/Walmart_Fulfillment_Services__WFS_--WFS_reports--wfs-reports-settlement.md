WFS Reports: Settlement

Last updated on Jul 9, 2025

Reading time: 3 min

* * *

In this guide:

- [Overview](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-settlement#overview)
- [Access this report](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-settlement#accessthisreport)
- [Dive into details](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-settlement#diveintodetails)
- [Understand report metrics](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-settlement#understandreportmetrics)

* * *

Overview

The Settlement report includes detailed itemization of all the WFS and MCS-specific fees charged (and credits issued) in a specific time period. For most WFS transactions, you'll find that this report delivers the most granular view.

**Access this report**

To access the Settlement report:

- Log in to Seller Center


- Select **WFS>Reports** from the left navigation pane


- Under Payments and Fees, select **Download** for the **Settlement** report


- Set your desired time period


- Select **Download** and open the report


### **Dive into details**

**How to use this report effectively**

- You can select your time period by settlement date, your closed statements or a custom set of dates


- Recent transactions will be available for review in your general payments statement once the statement period containing those transaction dates closes


- All fees and credits charged by WFS appear in the Settlement report


- Multichannel solution fees are included


- Storage fees are summarized


- The Settlement report is refreshed in real time, although different transaction types post at different intervals


**Important insights**

- Sorting the information by transaction type, reason code, and detail will give you the most information available to understand each row


- Before submitting a dispute for returns, check the Settlement report for more clarity


### **Understand report metrics**

|     |     |
| --- | --- |
| **Column name** | **Description** |
| ReportingPeriod\_StartDate | Starting date of the transactions covered in the report |
| ReportingPeriod\_EndDate | Ending date of the transactions covered in the report |
| Walmart.com Order # | Order number that the Walmart customer gets. One order number can be associated with multiple transactions. |
| Walmart.com Order Line # | Individual line items that are associated to the order number |
| Walmart.com PO # | Unique number assigned to a specific inbound order |
| Walmart.com PO Line # | Individual line items that are associated to the purchase order |
| Transaction type | Type of transaction on this line item that shows both debits and credits: <br>FulfillmentFee <br>ReturnProcessingFee <br>OffPeakSeasonStorageFee <br>PeakSeasonStorageFee <br>LongTermStorageFee <br>InboundTransportationFee <br>InventoryTransferFee <br>PrepServiceFee <br>RC\_InventoryDisposalFee <br>RC\_InventoryRTVFee <br>InventoryDisposalOrder <br>InventoryRemovalOrder <br>MultiChannelFulfillmentFee <br>Refund <br>Charge <br>DamageInWarehouse <br>LostInventory <br>FoundInventory |
| Reason code | Subtype of transaction <br>- For return shipping fee, we put return reason here <br>  <br>- For WFS fulfillment fees, we use this space to differentiate standard from big and bulky fulfillment for WFS <br>  <br>For MCS, we use this to differentiate 2 business day from 3-5 business day |
| Detail | Additional details on each line item are provided here, with more context on how we arrived at the rate. This includes: <br>- VirtualPackFeeReduction <br>  <br>- Oversize1 <br>  <br>  - Extra charge when one (or more) of these apply to your item:  A) Longest side greater than 48" and up to 96" B) Median side greater than 30" C) Longest side + girth greater than 105" and up to 130"<br>- Oversize2 <br>  <br>  - Extra charge when one (or more of these apply to your item: A) Longest side greater than 96" and up to 108" B) Longest side + girth greater than 130" and up to 165"<br>- Under$10 fee <br>  <br>- Hazmat <br>  <br>- Apparel |
| WFSReferenceID | Unique identifier in WFS (not used by seller) |
| Transaction Date/Time | Time the transaction took place |
| Qty | Unit quantity for this line item |
| Partner ID | Seller's unique ID within WFS |
| Partner GTIN | 14-digit number, including check digit, that's unique to the item. If it's less than 14 digits, add zeros at the beginning |
| Partner Item Name | Description of the item entered by the Seller on item setup |
| Category | Broad category for item |
| Sub Category | Further segmentation of the item from category defined |
| Net Payable | When positive, this represents the value the seller owes to WFS after discounts are applied; When negative, this represents the value WFS owes the seller. |
| CustomerOrderId | ID of an order the seller references |
| OrderChannelId | Channel Id is used to identify the channel that MCS orders came through |
| Original Amount | Pre-discount amount |
| Discount Savings | Discount reduction amount |
