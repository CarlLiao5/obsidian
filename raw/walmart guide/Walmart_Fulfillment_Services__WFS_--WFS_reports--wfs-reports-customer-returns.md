WFS reports: Customer returns

Last updated on Feb 9, 2026

Reading time: 3 min

* * *

In this guide:

- [Overview](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-customer-returns#overview)

- [Dive into details](https://marketplacelearn.walmart.com/guides/Walmart%20Fulfillment%20Services%20(WFS)/WFS%20reports/wfs-reports-customer-returns#diveintodetails)

* * *

Overview

As a WFS seller, you've had to utilize multiple reports to gain a complete understanding of customer returns. Now, have a comprehensive source of truth that merges customer returns and return-to-seller reports into one unified view. This provides greater clarity into the details of each return, including associated fees and reimbursements. You’ll be able to perform more accurate analysis, identify trends in customer returns, and pinpoint which items generate the highest volume.

## Access the report

1. Log in to Seller Center.
2. Navigate to the [**WFS Reports**](https://seller.walmart.com/wfs/reports) page.
3. Under **Sales**, select _Download_ for the **Customer returns** report.
4. Set your desired time period.
5. Select _Download_ and open the report.

## Dive into details

### How to use this report effectively

- Check the report to stay current on the latest return activity.
- Filter by **Return Channel** and **Fault** to separate customer-initiated issues from those attributed to sellers or third parties.
- Sort by **QTY** and **Return Reason** to surface high-volume SKUs and their top drivers, then inspect **Item Name / Description** for listing or quality fixes.
- Review **Keep-it (Y/N)** along with **Disposition Result** and **Condition Result** to quantify avoidable costs and fine-tune packaging, product details, and return preferences.

### Important insights

- For customer returns created after March 1, 2025, the report will include returned-to-seller (RTV) shipment information, including ship date, quantity shipped and tracking ID.
- **Return Reason** combined with **Fault** explain why items come back and who owns the issue, providing a roadmap for product, packaging or content improvements.
- **Disposition Result** and **Condition Result** signal whether a unit can be restocked, liquidated or returned to vendor, directly affecting recovery rates and margin.
- **Order Type** shows whether the return originated from standard marketplace flow or another channel, clarifying where process changes may be needed.
- **Date Processed in RC** and **RTV Ship Date** timestamps reveal cycle times inside the returns network, helping you measure operational efficiency and inventory recovery speed.

## Understand report metrics

| Column name | Description |
| --- | --- |
| Customer Order ID | Customer order identifier (also known as: Customer Order #, Order #, Sales Order ID, Sales Order #) |
| RMA Number | Return order identifier (also known as: Return Order #, Return Order ID) |
| RMA Line Number | Return order identifier for each item returned in an RMA (also known as: Return Order Line #) |
| RC Internal ID | Unique identifier for anything processed in a WFS Return Center (also known as: RCIID) |
| Item Name Description | Description of the item entered by the Seller on item setup |
| GTIN | Unique item identifier (also known as Global Trade Item Number) |
| QTY (returned) | Total number of items returned on the RMA Line Number in an RMA |
| Return Initiation Date | Timestamp at the time of the return creation event |
| Refund Status | Status of the refund issued to the customer |
| Return Delivery Tracking Status | Status of the returned item on its way from the customer to the Return Center |
| Return Reason | Customer selected reason of why the return was initiated |
| Fault | Classification of fault based on Return Reason |
| Return Channel | Method of return initiation |
| Keep-it (Y/N) | If keep-it was offered and accepted by the customer for the RMA Line Number |
| Date Delivered to RC | Date the return is confirmed delivered to the Return Center |
| Date Processed in RC | Date the return is received into ARC in the Return Center (timestamp correlates to the Disposition and Condition events) |
| Disposition Result (Latest) | Item disposition result after Return Center processing, based on RDM (WFS-fault) or Seller preferences (Seller-fault) |
| Condition Result (Latest) | Item condition result after Return Center grading process |
| RTV Ship Date | Date a shipment left the Return Center to the Seller (for RTV dispositioned items only) |
| RTV Tracking ID | Tracking ID for a shipment from the Return Center to the Seller (for RTV dispositioned items only) |
| RTV Carrier | Carrier for a shipment from the Return Center to the Seller (for RTV dispositioned items only) |
| Order Type | Sales channel the order was placed through |
| Shipped qty to seller | Number of items shipped from the Return Center to the Seller (for RTV dispositioned items only) |
