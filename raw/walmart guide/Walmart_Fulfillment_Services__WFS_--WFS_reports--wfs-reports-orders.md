WFS Reports: Orders

Last updated on Oct 29, 2025

Reading time: 2 min

Overview

As a WFS seller, it is beneficial to track and validate the customer orders that drive the Gross Merchandise Value (GMV) of your business.  You can use this report to track every order detail and conduct analysis based on this granular data.

### Access this report

- Log in to Seller Center

- Select **WFS>Reports** from the left navigation pane

- Under Sales, select **Download** for the **Orders** report

- Set your desired time period

- Select **Download** and open the report

### Dive into details

#### How to use this report effectively

- The data for this report will be updated at 4:30pm PST, and will go back 4 months.
- The report can be downloaded for data from the past 24 hours, last 7 days, last 30 days or a custom range of dates.

#### Important insights

- Since this report is by order/by item, you can use the data to see trends by item across orders.
- The data includes the destination location, so trending for geographic areas can be analyzed to help with better product placement.

### Understand report metrics

| **Column name** | **Description** |
| --- | --- |
| ORDER\_PLACED\_DT | This date will signify when the order was placed by the customer |
| SALES\_ORDER\_NUM | Order number that the Walmart customer gets. One order number can be associated with multiple transactions. |
| PURCHASE\_ORDER\_NUM | Unique number assigned to a specific inbound order |
| GTIN | 14-digit Global Trade Item Number, including check digit, that's unique to the item. If it's less than 14 digits, add zeros at the beginning |
| VENDOR\_SKU | Vendor Sku Id is a unique item identifier provided during item setup, that represents the item purchased by the customer |
| BUNDLE\_ITEM\_ID | Sellable item ID, when an item is part of a multi-box bundle.  If the item is not a multi-box item, this column will display "N/A". |
| WMT\_ITEM\_ID | Unique item identifier associated with displaying on marketplace.  The customer will see this number when ordering. |
| ITEM\_NAME | Description of the item entered by the Seller on item setup |
| QUANTITY | Count of the item purchased with this order |
| GMV\_AMT | Gross Merchandise Value (GMV) of the item that the customer ordered (unit price times units sold) |
| ORDER\_STATUS | Depending on when the report was last updated, an order can be in one of the following status':   <br>1) Acknowledged >> order is created but has not yet shipped<br>2) Shipped >> order has a shipped date <br>3) Delivered >> actual delivery date is available<br>5) Supplier Rejected >> when there is a date indicating the item on the order was cancelled by the seller<br>6) Cancelled due to backorder >> when there is a backorder date for an item, and the item on the order was cancelled |
| SHIPPED\_DATE | The date that the item left the FC location to be shipped, converted to PST |
| SHIPPED\_FROM | The FC location that the item was shipped from to the customer |
| SHIP\_TO\_CITY | The customer's city defined at purchase |
| SHIP\_TO\_ST | The customer's state defined at purchase |
| SHIP\_TO\_ZIP\_CD | The customer's zip code defined at purchase |
