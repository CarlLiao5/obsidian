# 降 Bid 反效果案例分析

> 真实案例：降 bid 后 Spend 反增 45%，ROAS 下降 22%——一个被算法反噬的典型场景。

## 案例背景

**基础知识参考**：
![[theory/Sponsored-Products--Campaign-bidding-strategies#出价策略类型]]

| 维度 | 降 bid 前（上个月） | 降 bid 后（本月） | 变化 |
|------|---------------------|-------------------|------|
| 月 Spend | $3,200 | $4,640 | **+$1,440 (+45%)** |
| CPC | $0.78 | $0.70 | **-$0.08 (-10%)** |
| 月点击量 | ~4,100 | ~6,600 | **+61%** |
| 转化率 | 9.48% | 7.32% | **-22.8%** |
| ROAS | 3.11 | 2.44 | **-21.5%** |
| 月收入 | $9,952 | $11,322 | +$1,370 |
| 日预算上限 | $150 | $150 | 不变 |

## 问题现象

运营者降 bid 的初衷是**控制花费**，但结果适得其反：
- Spend 没有降低，反而增加了 $1,440/月
- 转化率从 9.48% 下降到 7.32%
- ROAS 从 3.11 下降到 2.44

表面看是"广告效果变差"，实际上根源不在广告本身。

## 核心原因：每日预算耗尽时间点前移

```
降 bid 前：
  $0.78 CPC × ~137 次/天 = $107/天
  → 下午 2-3 点耗尽 $150 预算
  → 晚间高转化时段仍有广告展示（剩余 $43 预算覆盖核心时段）

降 bid 后：
  $0.70 CPC × ~214 次/天 = $150/天
  → 上午 10-11 点提前耗尽 $150 预算
  → 晚间 10+ 小时完全没有广告展示
```

### 算法反噬的机制

```
降低 bid → CPC 下降 10%（$0.78 → $0.70）
         → 同样 $150 预算买到更多点击（+61%）
         → 预算更快耗尽
         → 广告展示时间窗口大幅缩短
         → 丢失晚间高意向流量
         → 整体流量质量下降
         → 转化率下降
         → ROAS 下降
```

**关键洞察**：降 bid 降低的是**单次点击成本**，但同时也会改变**广告的展示时间分布**，而后者往往对转化率的影响更大。

## 影响分析

### 流量质量分层

| 时段 | 转化率特征 | 降 bid 后的损失 |
|------|-----------|----------------|
| 上午（6-12点） | 较低，浏览型流量为主 | 覆盖较多（提前耗尽前） |
| 下午（12-18点） | 中等 | 部分覆盖 |
| 晚间（18-24点） | **最高**，购买意向最强 | **完全丢失** |

晚间时段的访客通常是**主动购物的高意向买家**，转化率比白天高一倍以上。失去这个时段，是转化率下降的主要原因。

### 财务影响

```
本月 vs 上月：
  多花 $1,440 spend
  多赚 $1,370 收入
  净亏损：$70/月（还不算毛利率损耗）

如果维持原 bid（$0.78）和原 spend（$3,200）：
  预估收入 ≈ $9,952（接近上月水平）
  实际收入 ≈ $11,322
  损失 ≈ $1,370 - $70 = $1,300+ 潜在收入/月
```

## 诊断方法

当出现"降 bid 后 spend 反增 / 转化率下降"时，按以下步骤诊断：

### 第一步：确认 Cap-out 时间
在广告平台查看广告活动的**展示时间分布**（Hourly Impression Share 如果有的话）。如果发现降 bid 后广告总是**早于下午就停止展示**，说明是预算耗尽时间前移的问题。

### 第二步：检查日预算与实际 spend 的比值
| 比值 | 含义 | 应对 |
|------|------|------|
| 实际 spend < 日预算 × 28 天 | 预算未耗尽，流量正常 | 检查 bid 是否过高 |
| 实际 spend ≈ 日预算 × 30+ 天 | **预算提前耗尽** | 降 bid 策略失效，需调整 |
| 实际 spend > 日预算 × 30 天 | 超出预算上限 | 说明有超额消费，需检查[[theory/Out-of-budget-features]] |

### 第三步：对比 Page Type 分布
从 [[theory/All-Keywords]] 报告下载数据，对比 Browse 页 vs 搜索结果页的转化率。如果 Browse 页流量占比增加，说明广告从搜索结果首页退到了 Browse 页。

## 修复方案

### 方案一：同步降低日预算（推荐）

如果不想改变广告位置策略，可以**按比例降低日预算**，保持原有的展示节奏：

```
计算公式：
  原日均 spend = $3,200 / 30 = $107/天
  降 bid 后的目标日均 spend = $107/天（不变）
  
建议操作：
  将日预算从 $150 降到 $107
```

这样：
- CPC 维持低位（$0.70）
- 日 spend 控制在 $107
- 广告全天展示（不会提前耗尽）
- 牺牲的是点击量（从 ~6,600 降到 ~4,100），但保住晚间高转化流量

### 方案二：把 bid 加回去

如果目标是**恢复 ROAS 和转化率**，建议：

```
第一步：先把日预算降到 $107
第二步：逐步把 bid 加回 $0.78-0.80（每次 +10-15%）
第三步：观察 3-5 天后检查 Cap-out 时间是否恢复正常
第四步：如果 ROAS 回到 3.0+，维持当前设置
```

加回 bid 的节奏建议参考 [[theory/Sponsored-Products--Campaign-bidding-strategies]]。

### 方案三：使用 Target ROAS 出价

**参考**：![[theory/Sponsored-Products--Target-ROAS-bidding]]

如果希望系统自动平衡 spend 和转化，可以使用 [[theory/Sponsored-Products--Target-ROAS-bidding|Target ROAS 出价策略]]：
- 设置目标 ROAS 为 3.0（接近历史最优值）
- 系统自动在流量高峰时段多出价，低质量时段降低出价
- 无需手动频繁调整 bid

## 核心教训

> **降低 bid 不等于节省花费。**

在沃尔玛的第二竞价机制下（参考 [[practice/Algorithm-transition--Amazon-COSMO-vs-Walmart-second-price-auction]]），降 bid 会同时影响：
1. **单次点击成本**（降低）
2. **展示时间分布**（缩短）
3. **流量质量**（下降）
4. **整体转化效率**（恶化）

正确的控花费方式是：**先确保日预算与实际 spend 匹配，再决定是否调整 bid。**
