## 第一章 前置知识（平台认知）

### 1.1 沃尔玛平台概述

Walmart Marketplace 是沃尔玛旗下的第三方 marketplace，卖家在平台上架商品，由沃尔玛用户提供购买流量。与亚马逊的主要差异：

| 对比维度 | 亚马逊    | 沃尔玛                         |
| ---- | ------ | --------------------------- |
| 佣金率  | 6%–17% | 6%–15%                      |
| 广告竞价 | 第一价格拍卖 | **第二价格拍卖**（你出的价格通常低于实际扣费）   |
| 仓储服务 | FBA    | WFS（Fulfillment by Walmart） |
| 流量体量 | 全球最大   | 美国第二大，正快速增长                 |
| 入驻门槛 | 较高     | 相对较低，但对绩效要求严格               |

### 1.2 关键概念速查

| 术语             | 含义                                      |
| -------------- | --------------------------------------- |
| **SKU**        | Stock Keeping Unit，库存量单位，每个商品一个唯一编号     |
| **UPC / GTIN** | 商品条形码，用于识别商品身份，新品上架必备                   |
| **FBM**        | Fulfilled by Merchant，卖家自配送             |
| **WFS**        | Walmart Fulfillment Services，沃尔玛仓储配送服务  |
| **Buy Box**    | 购物车，当多个卖家售卖同一商品时，赢得 Buy Box 的卖家获得主要购买流量 |
| **VTR**        | Valid Tracking Rate，有效追踪率，需保持 ≥ 99%     |
| **ODR**        | Order Defect Rate，订单缺陷率，需保持在 < 1%       |

### 1.3 卖家中心（Seller Center）界面概览

登录 [Walmart Seller Center](https://seller.walmart.com)，左侧导航栏包含以下核心模块：

| 模块 | 功能说明 |
|------|----------|
| **Home** | 账户总览、绩效摘要、待处理事项 |
| **Catalog** | 商品目录管理（添加商品 / 编辑 / 批量上传） |
| **Orders** | 订单管理（查看 / 处理 / 追踪） |
| **Inventory** | 库存管理（WFS 库存 / 自配送库存） |
| **Advertising** | 广告活动管理（Sponsored Products） |
| **Reports** | 数据报告（销售 / 广告 / 库存） |
| **Settings** | 账户设置（业务信息 / 配送模板 / 退货规则） |
| **Performance** | 卖家绩效仪表盘 |

> 💡 **新人操作提示**：第一次登录后，优先检查 Settings → Business Information 确保公司信息无误，这是收款和税务的前置条件。

## 第二章 商品上架（Item Setup）

### 2.1 商品识别符选择

每个商品在 Walmart 上架前，必须先确定商品识别符类型：

| 识别符类型 | 适用商品 | 来源 |
|------------|----------|------|
| **GTIN / UPC** | 绝大多数商品，从正规渠道购买或由制造商提供 | GS1 官网购买 |
| **ISBN** | 图书类商品 | 出版社或 ISBN 机构 |
| **EAN** | 欧洲商品，部分国际品牌 | 制造商提供 |

⚠️ **踩坑点**：GTIN / UPC 必须在 GS1 官方注册购买的才有效，从第三方渠道购买的未注册条码会导致商品审核被拒或后续被下架。

### 2.2 上架方式对比

| 上架方式 | 适用场景 | 操作入口 | 复杂度 |
|----------|----------|----------|--------|
| **Create a new item** | 首次创建新品牌 / 新品 | Catalog → Add Item → Create new | 中 |
| **Match an existing item** | 沃尔玛目录已有同款商品，只需匹配 | Catalog → Add Item → Match existing | 低 |
| **Bulk upload** | 大量商品同时上传 | Catalog → Add Item → Upload spreadsheet | 高 |

### 2.3 新品上架完整流程（Create a new item）

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 进入 Add Item 页面 | Catalog → Add Item → Create new |
| 2 | 选择商品类目 | 输入商品关键词搜索匹配类目，确认 Category |
| 3 | 填写商品识别符 | 输入 UPC / GTIN，系统自动匹配商品信息 |
| 4 | 填写商品基本信息 | 品牌 / 型号 / 规格 / 包装数量 |
| 5 | 上传商品图片 | 主图必须白底、无水印、无其他品牌 logo |
| 6 | 填写商品描述 | 标题 / 五点描述 / 详细描述 / 关键词 |
| 7 | 设置价格和库存 | 售价（需符合价格领先规则）/ 库存数量 |
| 8 | 选择配送方式 | FBM（自配送模板）或 WFS（如已开通） |
| 9 | 提交审核 | 等待沃尔玛审核，审核周期通常 1–3 个工作日 |

### 2.4 商品内容优化五要素

#### ① 标题（Product Title）

| 标准 | 说明 |
|------|------|
| 长度 | 建议 50–150 个字符 |
| 格式 | 品牌 + 核心词 + 规格 / 型号 + 数量（如有）|
| 禁止 | 促销词（Sale / Free shipping）、全大写、无关关键词 |

#### ② 主图（Main Image）

| 标准 | 要求 |
|------|------|
| 背景 | 纯白底（RGB 255,255,255）|
| 尺寸 | 建议 2000px 以上，最小 500px |
| 内容 | 商品占比 ≥ 85%，无水印 / 无文字 / 无其他品牌 |
| 格式 | JPG 或 PNG |

#### ③ 五点描述（Key Product Features）

| 要素 | 内容 |
|------|------|
| Bullet 1 | 核心卖点和产品名称 |
| Bullet 2 | 规格参数（尺寸 / 材质 / 颜色 / 重量）|
| Bullet 3 | 适用场景或使用方法 |
| Bullet 4 | 包含配件说明（如有）|
| Bullet 5 | 品牌或质量保证信息 |

#### ④ 详细描述（Product Description）

- 补充五点描述未覆盖的内容
- 使用 HTML 格式（如有）增强可读性
- 包含使用说明、注意事项、保修信息

#### ⑤ 关键词（Search Terms）

| 类型 | 说明 | 数量上限 |
|------|------|----------|
| 通用关键词 | 与商品相关但标题未覆盖的搜索词 | — |
| 长尾关键词 | 更具体的场景词或问题词 | — |

### 2.5 上架后检查清单

在商品审核通过后，请立即执行以下检查：

- [ ] 商品是否在线（状态为 "Active"）
- [ ] 价格是否符合预期
- [ ] 库存数量是否正确
- [ ] 主图是否正常显示
- [ ] Buy Box 归属是否正常（如有竞品）
- [ ] 在卖家中心搜索商品名称，确认可正常找到

⚠️ **踩坑点**：审核通过后不要急于关闭页面，请立即检查商品是否正常展示。如发现价格错误或图片上传失败，需立即编辑修改。

## 第三章 配送与履约设置

### 3.1 FBM vs WFS 对比

| 维度 | FBM（自配送）| WFS（仓储配送）|
|------|--------------|----------------|
| **配送时效** | 3–7 个工作日 | 次日达 / 两日达 |
| **配送费用** | 自行与物流商结算 | 按 WFS 费率表结算 |
| **Buy Box 竞争力** | 较弱 | 较强（配送时效优势）|
| **适合阶段** | 新手测试阶段 / 重量级大件 | 有一定销量后提升竞争力 |

### 3.2 自配送配置（FBM）

#### 路径：Settings → Shipping Settings

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 创建配送模板 | 设置配送区域和时效 |
| 2 | 设置默认发货延迟时间 | Walmart 要求 ≤ 1 个工作日（工作日不含周末和节假日）|
| 3 | 配置承运商 | 选择支持的物流服务商（USPS / UPS / FedEx）|
| 4 | 关联配送模板到商品 | 上架时选择对应配送模板 |

⚠️ **踩坑点**：发货延迟时间（Shipment Latency）设置过长会影响 OTD（准时送达率）。建议根据实际备货时间合理设置，默认不要超过 2 天。

### 3.3 WFS 入驻概览

WFS 适合有一定销量积累、希望提升配送竞争力的卖家：

| 入驻条件 | 说明 |
|----------|------|
| Walmart 账户良好 | 绩效指标达标，无严重违规 |
| 有一定销量基础 | 通常建议月销达到一定门槛再考虑 WFS |
| 商品符合 WFS 要求 | 非危险品、非超大件、可标准化包装 |

#### WFS 核心费用参考

| 费用类型 | 常规期 | 旺季（10–12 月）|
|----------|--------|-----------------|
| 仓储费 | $0.75 / 立方英尺 / 月 | $2.40 / 立方英尺 / 月 |
| 配送费（≤1lb）| $2.45 | 同 |

## 第四章 广告投放入门（Sponsored Products）

### 4.1 广告基础认知

Sponsored Products 是 Walmart Connect 提供的按点击付费（PPC）站内广告，帮你把商品展示在搜索结果页或商品详情页。

| 特点 | 说明 |
|------|------|
| **计费方式** | 按点击付费（CPC），展示免费 |
| **竞价机制** | 第二价格拍卖（你支付的金额通常低于出价） |
| **展示位置** | 搜索结果页（Search In-grid）、商品详情页（My Items）|
| **适用阶段** | 新品破零、爆款维护、清理库存 |

### 4.2 两种活动类型

| 类型                  | 说明          | 推荐场景           |
| ------------------- | ----------- | -------------- |
| **自动活动（Automatic）** | 系统自动匹配相关搜索词 | 新手上路、快速测试关键词方向 |
| **手动活动（Manual）**    | 你自行设置关键词和出价 | 精细化运营、有明确关键词策略 |

### 4.3 新建广告活动完整流程

#### 第一步：创建活动

| Step | 操作                 | 说明                       |
| ---- | ------------------ | ------------------------ |
| 1    | 进入 Advertising 模块  | 点击左侧导航 Advertising       |
| 2    | 点击 Create Campaign | 进入活动创建页面                 |
| 3    | 填写活动名称             | 建议命名格式：[产品线]_[活动类型]_[日期] |
| 4    | 设置日预算              | 建议从 $10–$20 开始测试         |
| 5    | 选择出价策略             | 新手推荐 Fixed Bidding（固定出价） |
| 6    | 设置活动周期             | 设置开始日期，建议至少运行 14 天再评估    |

#### 第二步：创建广告组

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 为广告活动添加广告组 | 一个活动可包含多个广告组 |
| 2 | 选择投放商品 | 挑选要推广的 SKU |
| 3 | 选择活动类型 | 自动或手动 |

#### 第三步（手动活动）：添加关键词

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 输入核心关键词 | 建议参考五点描述和搜索建议 |
| 2 | 选择匹配类型 | Broad / Phrase / Exact（精准度递增）|
| 3 | 为每个关键词设置出价 | 建议从 $0.50–$1.00 开始 |

#### 第四步：设置展示位置出价倍数（可选）

| 展示位置 | 出价倍数建议 | 说明 |
|----------|-------------|------|
| Search In-grid | 1.0x（基准）| — |
| My Items | 0.5x–0.8x | 浏览流量转化率通常低于搜索流量 |

#### 第五步：提交并激活

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 检查所有设置无误 | 确认预算 / 出价 / 关键词 |
| 2 | 提交活动 | 系统审核（通常几分钟内通过）|
| 3 | 监控数据 | 3–5 天后开始有足够数据进行分析 |

### 4.4 核心广告指标解读

| 指标 | 全称 | 含义 | 健康范围 |
|------|------|------|----------|
| **CTR** | Click-Through Rate，点击率 | 曝光中有多少比例产生了点击 | > 0.5% |
| **CPC** | Cost Per Click，单次点击成本 | 每次点击你实际支付的费用 | 符合预期 ACOS |
| **ACOS** | Advertising Cost of Sales，广告销售占比 | 广告花费 / 广告带来的销售额 | < 30%（视类目而定）|
| **ROAS** | Return on Ad Spend，广告支出回报率 | 广告带来的销售额 / 广告花费 | > 3x |
| **Impressions** | 展示次数 | 广告被看到的次数 | — |

⚠️ **踩坑点**：不要只盯 ACOS。新品期应优先关注 **CTR** 和 **转化量**（有足够的销量才能积累自然排名），等到关键词排名稳定后再优化 ACOS。

### 4.5 新品期广告策略建议

| 阶段 | 目标 | 策略 |
|------|------|------|
| Day 1–14 | 积累数据 | 自动活动 + 固定出价，预算 $10–$15/天 |
| Day 15–30 | 找到高效词 | 分析点击和转化数据，找出 3–5 个高效关键词 |
| Day 30+ | 精细化运营 | 将高效关键词单独建立手动活动，提高出价 |

⚠️ **踩坑点**：不要在数据不足时（< 7 天或 < 100 次点击）频繁调整出价。数据量太少时，调整可能是噪音而非信号。

## 第五章 日常运营

> ⚠️ 本章提供运营框架和检查流程，详细 SOP 清单请参阅 Operations SOP   Daily and Weekly Tasks。不要只靠记忆，每次操作前对照 SOP 清单执行。

### 5.1 每日启动检查流程（3 步法）

#### 第一步：查看数据总览

- 打开 Seller Center Home，查看今日待处理事项数量
- 检查订单模块，确认是否有新订单未处理

#### 第二步：处理预警事项

按以下优先级处理：

| 优先级 | 事项 | 操作时限 |
|--------|------|----------|
| 🔴 高 | 站内消息未回复 | 当日（24 小时内）|
| 🔴 高 | 新退货请求 | 当日 |
| 🔴 高 | 绩效预警 | 当日排查原因 |
| 🟡 中 | 未发布商品（被下架）| 24 小时内修复 |
| 🟡 中 | Buy Box 丢失 | 排查原因，调整价格或库存 |
| 🟢 低 | 广告花费异常 | 当日内检查广告活动设置 |

#### 第三步：处理日常任务

- 确认当日发货订单已标记已发货
- 检查库存是否需要补货
- 记录当日核心数据（销量 / 销售额 / 广告花费）

### 5.2 每周复盘框架

| 复盘维度 | 具体内容 |
|----------|----------|
| **销售回顾** | 各 SKU 销量和销售额，是否达到预期 |
| **广告复盘** | ACOS / CTR / CPC 是否在健康范围 |
| **库存盘点** | 哪些 SKU 需要补货，缺货预警 |
| **退货分析** | 退货主要原因，识别系统性问题 |
| **竞品观察** | 主要竞品价格和促销动态 |

### 5.3 核心运营工具入口

| 工具 | 入口 | 用途 |
|------|------|------|
| 卖家记分卡 | Performance → Scorecard | 监控 ODR / OTD / VTR |
| 广告报告 | Advertising → Reports | 分析广告效果 |
| Buy Box 报告 | Reports → Buy Box | 查看 Buy Box 占有率 |
| 退货洞察 | Returns → Insights | 分析退货原因趋势 |
| 关键词规划器 | Growth → Keywords Planner | 调研搜索流量 |

## 第六章 订单与退货处理

### 6.1 订单处理全流程

```
收到订单 → 确认订单（ACK）→ 备货打包 → 发货并上传追踪号 → 标记已发货
```

| Step | 操作    | 时限要求    | 说明                                 |
| ---- | ----- | ------- | ---------------------------------- |
| 1    | 确认订单  | 即时或当日   | 进入 Orders → 找到新订单 → 点击 Acknowledge |
| 2    | 备货打包  | 按承诺时间   | 核实商品数量和包装                          |
| 3    | 发货    | 发货延迟时间内 | 选择物流商并打印标签                         |
| 4    | 上传追踪号 | 发货当日    | 在 Seller Center 上传有效追踪号            |
| 5    | 标记已发货 | 上传追踪号后  | 系统自动更新订单状态                         |

⚠️ **踩坑点**：追踪号必须真实有效，且在承运商系统可查询。无效追踪号会拉低 VTR，影响绩效评分。

### 6.2 有效追踪率（VTR）要求

| 指标 | 要求 | 计算方式 |
|------|------|----------|
| VTR | ≥ 99% | 有效追踪包裹数 / 总发货包裹数 |

⚠️ **踩坑点**：以下情况会被判定为**无效追踪**：
- 追踪号格式错误
- 承运商系统查不到物流记录
- 包裹未在承诺时效内发出
- 物流信息在 30 天内未更新

### 6.3 退货请求处理

#### 退货处理流程

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 收到退货请求通知 | 系统自动发站内消息通知 |
| 2 | 查看退货原因 | 区分：误购 / 质量问题 / 描述不符 / 物流损坏 |
| 3 | 审批退货请求 | 同意后买家将收到退货标签 |
| 4 | 等待买家退货 | 通常 5–10 个工作日 |
| 5 | 收到退货包裹 | 仓库或你亲自签收 |
| 6 | 检查退货商品状态 | 符合条件 → 退款；不符合 → 拒绝退款并说明原因 |
| 7 | 执行退款 | 在 Seller Center 操作退款 |

#### 退货原因分类与应对

| 退货原因    | 应对措施               |
| ------- | ------------------ |
| 质量问题    | 记录反馈给产品团队，考虑更换供应商  |
| 描述不符    | 立即优化 Listing 描述和图片 |
| 误购      | 优化标题和分类，减少误购可能     |
| 物流损坏    | 改进包装，考虑更换更可靠的物流商   |
| 尺寸/颜色不对 | 优化规格描述，提供更准确的尺码表   |

### 6.4 Keep It（退款不退货）规则

Walmart 提供 Keep It 功能，允许在特定情况下直接退款而不要求退货：

| 条件 | 说明 |
|------|------|
| 订单金额门槛 | 通常 ≤ $25（沃尔玛根据品类评估）|
| 适用原因 | 轻微描述不符 / 误购 / 部分损坏 |
| 配置入口 | Settings → Return Settings → Add Keep It Rule |

### 6.5 订单取消规则

| 情况 | 可否取消 | 时限要求 |
|------|----------|----------|
| 买家主动取消 | ✅ 可以 | 订单确认后 30 分钟内 |
| 库存不足 | ✅ 可以 | 需选择正确取消原因（Inventory unavailable）|
| 超过 30 分钟 | ❌ 需走特殊渠道 | 建议联系买家说明情况 |

⚠️ **踩坑点**：频繁取消订单（取消率 > 2%）会触发绩效警告甚至账户暂停。遇到缺货情况务必提前主动处理。

## 第七章 绩效达标（记分卡解读）

### 7.1 沃尔玛卖家记分卡四大指标

#### 路径：Performance → Scorecard

| 指标 | 全称 | 健康标准 | 预警阈值 | 超过后果 |
|------|------|----------|----------|----------|
| **ODR** | Order Defect Rate（订单缺陷率）| < 1% | 1%–2% | 严重违规警告 / 账户暂停 |
| **OTD** | On-Time Delivery（准时送达率）| > 97% | 95%–97% | 影响 Buy Box 竞争力 |
| **VTR** | Valid Tracking Rate（有效追踪率）| > 99% | 97%–99% | 账户绩效警告 |
| **TRTR** | Total Return Rate（总退货率）| 视类目而定 | — | 长期高退货可能影响Listing |

### 7.2 指标详解

#### ODR（订单缺陷率）

ODR 由以下三项构成：

| 组成项 | 计算方式 |
|--------|----------|
| 负面评价率 | 收到 1–2 星评价的订单占比 |
| 物品未收到率（INR）| 买家称未收到商品的订单占比 |
| 退货率 | 因描述不符或质量问题导致的退货占比 |

**修复建议**：
- 负面评价：及时在商品页公开回复，说明解决方案
- INR：确保追踪号真实有效，提前告知买家配送预期
- 质量退货：分析退货原因，改进产品或描述

#### OTD（准时送达率）

| 关键因素 | 说明 |
|----------|------|
| 发货延迟时间 | 从订单确认到发货的时长，必须 ≤ 1 个工作日 |
| 承运商承诺时效 | 选择的物流方式必须在 Walmart 规定时效内到达 |
| 节假日 | 节假日期间需提前调整发货延迟时间 |

#### VTR（有效追踪率）

保持 VTR ≥ 99% 的核心方法：

- 使用 Walmart 认可的承运商（USPS / UPS / FedEx）
- 确保追踪号格式正确且可查询
- 发货当日上传追踪号，不要延迟
- 每月检查 VTR 报告，识别问题包裹

### 7.3 绩效预警响应流程

```
指标低于健康标准
    ↓
立即排查原因（参考上表各项因素）
    ↓
制定改善计划（当天内完成）
    ↓
执行改善措施
    ↓
3–5 个工作日后再次检查指标
```

⚠️ **踩坑点**：绩效指标出现预警后不要拖延。沃尔玛的账户审查是自动触发的，持续不达标可能导致账户被暂停。

## 第八章 平台红线与合规要求

> ⚠️ **最重要的一条规则：遇到不确定的操作，请先查阅 Wiki 或请教同事，不要凭直觉操作。平台合规底线一旦触碰，轻则下架，重则封号。**

### 8.1 定价合规红线

| 规则 | 说明 | 违反后果 |
|------|------|----------|
| **价格领先（Price Leadership）** | 同一商品在 Walmart 的售价必须低于或等于在其他平台的售价 | 商品被下架 / 账户警告 |
| **价格一致性（Price Parity）** | 同一商品在 Walmart 平台内的价格必须保持一致（同一 SKU 不能有两个不同价格）| 商品被下架 |
| **最低价格保证（Lowest Price Guarantee）** | 部分类目要求提供最低价，沃尔玛会自动监控 | 被自动调价（部分情况）|

⚠️ **踩坑点**：即使你同时运营亚马逊和沃尔玛，也必须确保 Walmart 价格不高于其他平台。在大促前尤其注意检查价格一致性。

### 8.2 禁售商品类别

沃尔玛对以下商品类别有严格的准入限制，部分需要预先审批或无法销售：

| 禁售 / 限制类别 | 说明 |
|----------------|------|
| 食品（部分类目）| 保质期管理要求高，需预先审批 |
| 药品和保健品 | 需 FDA 认证 |
| 危险品（锂电池、压力罐等）| 需特殊认证和包装 |
| 侵权商品 | 仿牌、盗版、未经授权的品牌商品 |
| 二手商品 | 原则上 Walmart Marketplace 仅接受新品 |
| 枪支和武器 | 完全禁止 |

⚠️ **踩坑点**：如果你不确定某个商品是否可以在沃尔玛销售，**不要先上架再说**，请先查阅官方禁售清单或咨询账户经理。

### 8.3 知识产权保护

| 违规类型 | 说明 | 后果 |
|----------|------|------|
| 商标侵权 | 使用他人注册商标（如品牌 logo、品牌名）| 商品下架，账户警告 |
| 专利侵权 | 销售含有他人专利技术的产品 | 商品下架，可能面临法律诉讼 |
| 版权侵权 | 使用未经授权的图片、文字、视频内容 | 商品下架 |

**预防措施**：
- 只销售有权销售的品牌商品
- 图片使用自制图片或沃尔玛授权图片
- 标题和描述不要包含他人商标词

### 8.4 账户暂停与申诉

如果账户收到暂停通知：

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 查看暂停原因 | Walmart 会通过邮件和 Seller Center 发送详细原因 |
| 2 | 收集证据 | 整理订单记录、发货凭证、沟通记录 |
| 3 | 撰写申诉 | 说明原因分析 + 改善计划 + 预防措施 |
| 4 | 提交申诉 | 通过 Seller Center 提交，等待账户团队审核 |
| 5 | 等待结果 | 通常 3–5 个工作日有回复 |

⚠️ **踩坑点**：申诉时不要指责沃尔玛或甩锅给买家。账户团队要看的是你是否有清晰的自我反省和切实可行的改善计划。

### 8.5 合规自检清单

以下操作是红线，**绝对不要做**：

- [ ] 在沃尔玛以低于其他平台的价格销售同一商品
- [ ] 上架未注册 UPC / GTIN 的商品
- [ ] 使用未经授权的品牌图片或 logo
- [ ] 销售禁售类目商品
- [ ] 故意操纵评论或评分
- [ ] 创建多个账户规避绩效问题
- [ ] 在不确定时凭直觉操作（先查 Wiki 或问同事）
