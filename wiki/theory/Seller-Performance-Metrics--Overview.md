## OTD - On-Time Delivery（准时送达率）

### 定义

OTD 衡量订单在承诺发货/送达时间内完成履约的比例。沃尔玛将订单配送时间与承运商承诺时效进行比对，计算准时率。

### 计算公式

```
OTD = (准时送达订单数 ÷ 总订单数) × 100%
```

### 影响因素

| 因素 | 说明 |
|------|------|
| 发货延迟 | 超出发货承诺时间（Promise Date） |
| 承运商承诺时效 | 承运商系统中的配送时效设置 |
| 节假日 | 节假日前后物流积压 |
| 自然灾害 | 不可抗力导致的延误 |
| 偏远地区 | 部分邮编配送时效较长 |

### 达标标准

| 等级 | OTD 范围 |
|------|---------|
| 绿色（达标） | ≥ 97% |
| 黄色（预警） | 90% - 96.9% |
| 红色（不达标） | < 90% |

### 优化方法

1. **设置合理的发货承诺时间**：不要设置过短的备货期
2. **使用 WFS**：Fulfillment by Walmart 履约的 OTD 通常更稳定
3. **监控库存深度**：断货是 OTD 下降的主要原因
4. **提前处理节假日**：节假日前提前备货
5. **选择可靠的承运商**：追踪稳定性影响 OTD 计算

**参考**：`[[theory/simplified-shipping-settings-overview]]` — 发货设置与 OTD 保护机制

## VTR - Valid Tracking Rate（有效追踪率）

### 定义

VTR 衡量所有带有有效承运商追踪码的包裹占比。只有在承运商系统中产生实际扫描记录的追踪码才被视为"有效"。

### 计算公式

```
VTR = (有效追踪码包裹数 ÷ 使用追踪码的总包裹数) × 100%
```

### 达标标准

| 等级 | VTR 范围 |
|------|---------|
| 绿色（达标） | ≥ 99% |
| 黄色（预警） | 95% - 98.9% |
| 红色（不达标） | < 95% |

### 常见导致 VTR 下降的原因

| 问题 | 说明 |
|------|------|
| 承运商扫描缺失 | 包裹发出但未被承运商扫描 |
| 追踪码格式错误 | 使用了沃尔玛不认可的追踪码格式 |
| 虚拟追踪码 | 承运商预分配但未实际揽收的追踪码 |
| 承运商未接入沃尔玛 | 使用了未在沃尔玛注册的承运商 |

### 优化方法

1. **仅使用沃尔玛认可的承运商**：在 [[theory/Prohibited-content-and-products]] 和平台政策允许的承运商列表中选择
2. **追踪码上传及时**：发货时立即上传追踪码，不要事后补录
3. **验证追踪码有效性**：发出前在承运商官网确认有首条扫描记录
4. **清理无效追踪**：定期检查 VTR 报告，移除长期无扫描的追踪码
5. **小包发货使用带扫描的追踪服务**：避免使用平邮等无追踪服务

**参考**：`[[theory/Onboarding-Guide--New-Employee-Training]]` 第 386-396 行 — VTR 详细说明

## ODR - Order Defect Rate（订单缺陷率）

### 定义

ODR 是衡量订单质量的综合指标，由三部分组成：

```
ODR = 负面评价率 + INR率 + 退货率
```

### 三个组成部分

| 组成 | 定义 | 权重说明 |
|------|------|---------|
| **负面评价率** | 收到 1-2 星评价的订单占比 | 评价星级 ≤ 2 的订单 ÷ 总订单 |
| **INR 率**（Item Not Received） | 买家未收到商品但发起 Case 的比例 | INR Case 数 ÷ 总订单数 |
| **退货率** | 订单产生退货的比例 | 退货订单数 ÷ 总订单数 |

### 达标标准

| 等级 | ODR 范围 |
|------|---------|
| 绿色（达标） | < 1% |
| 黄色（预警） | 1% - 2% |
| 红色（不达标） | ≥ 2% |

### 优化方法

#### 负面评价率
- 主动发送索评邮件（合规范围内）
- 及时处理差评订单的客户诉求
- 优化商品详情页减少预期差导致的差评

#### INR 率
- 确保发货地址准确
- 使用有追踪的配送方式
- 设置合理的配送时效承诺
- 及时处理买家"未收到"的询问

#### 退货率
- 详情页描述准确，减少货不对板
- 优化包装减少运输损坏
- 提供清晰的安装/使用说明

**参考**：`[[theory/Onboarding-Guide--New-Employee-Training]]` 第 466-479 行 — ODR 详细说明

## TRTR - Total Return Rate（总退货率）

### 定义

TRTR 衡量总订单中产生退货的比例。与 ODR 中的退货率不同，TRTR 是独立的监控指标，侧重于退货规模而非缺陷原因。

### 达标标准

> **注意**：TRTR 无固定达标值，不同类目有不同的退货率基准线。

| 类目特征 | 预期 TRTR 范围 |
|---------|--------------|
| 服装/鞋类 | 10% - 20% |
| 电子产品 | 5% - 10% |
| 家居用品 | 5% - 15% |
| 食品/个护 | 1% - 5% |

### 分析维度

| 维度 | 说明 |
|------|------|
| 按 SKU 分析 | 识别高退货率的具体商品 |
| 按退货原因分析 | 货不对板/损坏/不需要等分类统计 |
| 按时间分析 | 是否季节性波动 |
| 按买家行为分析 | 重复退货买家的识别 |

### 优化方法

1. **SKU 级别退货率监控**：对 TRTR > 类目均值的 SKU 优先优化
2. **退货原因分析**：从退货理由中提取改进方向
3. **详情页准确性**：图片、尺寸、规格描述与实物一致
4. **包装升级**：减少运输损坏
5. **买家教育**：在详情页提供清晰的使用说明

**参考**：`[[theory/Order_management--Returns_&_refunds]]` — 退货政策与处理流程
